1. Blocked on a Simple Problem:

function capitalize(string) { let capitalize = string.split(" ").map( letter => letter.charAt(0).toUpperCase() + letter.slice(1)); return capitalize.join(" "); } When I logged the result to the console, I was able to get the first letter of each word capitalized, but I didn’t know how to return them as a full sentence. Initially, I thought about using something like letter.charAt(0).toUpperCase() + letter.charAt(1) + ... and then Googled how to combine the letters. That’s when I learned about the `slice` method and how to use it.

2. Elegantly Solving a Problem:

 function enterScores(gradebook, students, scores) { for (let i = 0; i < students.length; i++) { gradebook[students[i]] = { testScores: scores[i] }; } return gradebook; }
At first, I was thinking of a more complicated approach (like using two for loops to add the data to the array). But then I realized that the indexes of the arrays matched exactly. So, I decided to combine the logic into a single loop. It was initially tricky to think of { testScores: scores[i] }, but once I understood the structure of arrays and objects, it became much easier to find the solution.

3. Reflecting on Problem-Solving Techniques:

**Pseudocode:** Before writing any code, I take some time to think and write pseudocode first. This helps me structure the algorithm. I always use this technique. **10/10** **Trying something:** If the code I wrote doesn’t work, I try a different approach before Googling. **10/10** **Rubber ducky method:** When I’m really stuck, I explain the problem to myself from the ground up. During this process, I often find the answer. **9/10** **Reading error messages:** Absolutely essential. I always read error messages to guide my debugging. **10/10** **Console logging:** I always log results to the console. Seeing the output helps me identify and fix issues. **10/10** **Googling:** A lifesaver! I rely on it a lot. **10/10** **Asking your peers for help:** So far, I’ve been able to solve problems before asking peers, but I’ll definitely ask if I’m really stuck. **Asking coaches for help:** Similar to peers, I haven’t needed to ask coaches yet, but I will if necessary. **Improving your process with reflection:** Reflecting is really helpful, especially after spending a long time on a problem. I always redo problems I struggled with. **10/10**

4. Reflecting on Reluctance to Ask for Help:

I think I’ve had the mindset that I must solve problems by myself. Unless I can’t solve them within two days (which is my personal deadline—haha), I don’t usually ask for help. However, I’ve realized that I should ask for help sooner than that. Next time, I’ll make sure to ask for help more quickly.
